在 IM 系统的 WebSocket 协议中，**类型（`type`）** 用于区分消息的**大类**（如系统控制、会话管理、消息业务等），**子命令（`cmd`）** 用于在大类下细分**具体功能**（如发送消息、删除会话）。以下是**最常用的类型与子命令列表**，覆盖核心场景（认证、会话、消息、联系人、系统通知等），结构清晰且贴近实际开发：


### **一、核心类型与子命令总览（表格版）**
| **类型（`type`）** | **子命令（`cmd`）** | **方向**               | **用途说明**                                  | **示例数据（`data` 字段）**                          |
|---------------------|---------------------|------------------------|-----------------------------------------------|------------------------------------------------------|
| **`SYSTEM`**        | `AUTH_REQ`           | 客户端→服务端          | 连接后发起身份认证（首次必发）                | `{ "token": "xxx", "userId": "u123", "deviceId": "d456" }` |
|                     | `AUTH_RESP`          | 服务端→客户端          | 返回认证结果（成功/失败）                      | `{ "code": 0, "msg": "成功", "sessionId": "sess_789" }` |
|                     | `HEARTBEAT`          | 客户端→服务端          | 心跳包（维持连接，防断开）                    | `{ "seq": 123 }`（递增序列号）                        |
|                     | `HEARTBEAT_ACK`      | 服务端→客户端          | 心跳响应（确认存活）                          | `{ "seq": 123, "serverTime": 1717267200 }`            |
|                     | `KICKOUT`            | 服务端→客户端          | 被踢下线通知（如异地登录）                    | `{ "reason": "账号在其他设备登录" }`                  |
|                     | `ERROR`              | 服务端→客户端          | 错误通知（如认证失败、参数错误）              | `{ "code": 401, "msg": "无效token" }`                 |
| **`CONVERSATION`**  | `FETCH_LIST`         | 客户端→服务端          | 拉取会话列表（分页）                          | `{ "page": 1, "size": 20, "unreadOnly": true }`       |
|                     | `DELETE`             | 客户端→服务端          | 删除指定会话（如清空聊天记录）                | `{ "convId": "conv_abc123" }`                         |
|                     | `UPDATE`             | 客户端→服务端          | 更新会话属性（置顶、免打扰、修改备注）        | `{ "convId": "conv_abc123", "top": true, "mute": false }` |
|                     | `SYNC`               | 服务端→客户端          | 同步会话变更（如新会话创建、属性更新）        | `{ "convId": "conv_abc123", "action": "create" }`     |
| **`MESSAGE`**        | `SEND`               | 客户端→服务端          | 发送消息（文本、图片、文件等）                | `{ "to": "u456", "contentType": "TEXT", "content": "Hi" }` |
|                     | `RECALL`             | 客户端→服务端          | 撤回消息（限时可撤回）                        | `{ "msgId": "msg_xyz789", "convId": "conv_abc123" }`   |
|                     | `ACK`                | 服务端→客户端          | 消息回执（对方已读/未读）                      | `{ "msgId": "msg_xyz789", "status": 1 }`（1=已读）    |
|                     | `NEW`                | 服务端→客户端          | 推送新消息（对方发送的消息）                  | `{ "msgId": "msg_new456", "from": "u456", "content": "Hello" }` |
|                     | `HISTORY_FETCH`      | 客户端→服务端          | 拉取历史消息（按会话/时间范围）                | `{ "convId": "conv_abc123", "startTime": 1717267200, "limit": 50 }` |
| **`CONTACT`**        | `ADD`                | 客户端→服务端          | 添加好友/群组                                 | `{ "userId": "u789", "remark": "同事" }`              |
|                     | `LIST`               | 客户端→服务端          | 获取联系人列表（好友/群组）                    | `{ "type": "FRIEND", "page": 1 }`                     |
|                     | `DELETE`             | 客户端→服务端          | 删除联系人（好友/群组退出）                    | `{ "contactId": "u789", "type": "FRIEND" }`           |
|                     | `UPDATE_REMARK`      | 客户端→服务端          | 修改联系人备注                                | `{ "contactId": "u789", "remark": "新备注" }`          |
| **`NOTIFICATION`**   | `SYSTEM`             | 服务端→客户端          | 系统通知（版本更新、活动公告）                | `{ "title": "新版本", "content": "点击更新" }`        |
|                     | `REMIND`             | 服务端→客户端          | 提醒类通知（如@提及、任务到期）                | `{ "type": "@", "convId": "conv_abc123", "from": "u456" }` |
| **`FILE`**           | `UPLOAD_REQ`         | 客户端→服务端          | 请求上传文件（获取上传地址/凭证）              | `{ "fileName": "doc.pdf", "size": 102400 }`           |
|                     | `UPLOAD_PROGRESS`    | 服务端→客户端          | 推送上传进度（百分比）                        | `{ "fileId": "file_xyz", "progress": 80 }`            |
|                     | `DOWNLOAD_REQ`       | 客户端→服务端          | 请求下载文件（获取下载地址）                  | `{ "fileId": "file_xyz" }`                            |


### **二、关键类型与子命令详解（按场景分类）**
#### **1. 系统控制（`SYSTEM`）：连接与基础管理**
- **核心作用**：处理连接生命周期（认证、心跳、错误、踢下线）。
- **必选子命令**：
    - `AUTH_REQ/AUTH_RESP`：连接后**立即执行**，未完成认证前拒收业务消息。
    - `HEARTBEAT/HEARTBEAT_ACK`：每 30 秒发送一次心跳，维持长连接（防 NAT 超时）。
- **示例流程**：  
  客户端连接 → 发送 `AUTH_REQ` → 服务端返回 `AUTH_RESP`（成功则 `isAuthenticated=true`）→ 定期发送 `HEARTBEAT` → 服务端响应 `HEARTBEAT_ACK`。


#### **2. 会话管理（`CONVERSATION`）：聊天窗口相关**
- **核心作用**：管理用户与用户/群组的“对话容器”（如单聊会话、群聊会话）。
- **常用子命令**：
    - `FETCH_LIST`：打开 IM 首页时拉取会话列表（带未读数、最后一条消息摘要）。
    - `UPDATE`：用户手动置顶会话、开启免打扰（如“@我时才提醒”）。
    - `SYNC`：服务端主动推送会话变更（如对方新建群聊，当前用户被拉入）。


#### **3. 消息处理（`MESSAGE`）：核心业务数据**
- **核心作用**：发送/接收/管理具体聊天内容（文本、图片、表情、文件等）。
- **关键子命令**：
    - `SEND`：客户端发送消息（需携带 `contentType` 区分类型：`TEXT`/`IMAGE`/`VOICE`/`FILE`）。
    - `NEW`：服务端推送新消息（对方发送后，实时推送到当前用户）。
    - `ACK`：消息回执（对方已读时，服务端推送 `ACK` 给发送方，更新 UI 为“已读”）。
    - `RECALL`：撤回消息（需校验撤回时限，如 2 分钟内）。


#### **4. 联系人管理（`CONTACT`）：社交关系**
- **核心作用**：管理用户的好友、群组等社交关系。
- **常用子命令**：
    - `ADD`：发送好友申请（需对方同意，同意后自动创建会话）。
    - `LIST`：获取好友列表（按分组：家人/同事/同学）或群组列表。


#### **5. 系统通知（`NOTIFICATION`）：非业务提醒**
- **核心作用**：推送与聊天无关的系统级提醒（如版本更新、账号安全通知）。
- **与 `MESSAGE` 的区别**：`NOTIFICATION` 不进入会话列表，直接弹窗或显示在通知栏。


### **三、设计思路总结**
1. **类型（`type`）**：按**业务域**划分大类（系统、会话、消息、联系人），避免单一类型下子命令过多。
2. **子命令（`cmd`）**：用**动词+名词**命名（如 `FETCH_LIST`/`DELETE_CONV`），直观反映功能。
3. **方向明确**：标注“客户端→服务端”或“服务端→客户端”，避免双向混淆（如 `SEND` 是客户端发，`NEW` 是服务端推）。
4. **扩展性**：预留 `CUSTOM` 类型或 `EXT_` 前缀子命令（如 `CUSTOM/EXT_PAY`），支持业务定制。


通过以上列表，可快速搭建 IM 协议的消息框架，覆盖 90% 以上的常见场景。实际开发中，可根据业务需求增删子命令（如增加 `GROUP_CREATE` 子命令到 `CONVERSATION` 类型管理群聊创建）。
